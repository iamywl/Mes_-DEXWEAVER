# 주간보고 (2026년 2월 6일)

## 🏗️ 시스템 아키텍처 (전체 구조도)
스마트 팩토리 MES 프로젝트는 여러 구성 요소가 유기적으로 연결되어 동작하는 복잡한 시스템입니다. 사용자 경험을 담당하는 프론트엔드부터 실제 데이터 처리와 저장을 담당하는 백엔드 및 데이터베이스까지, 모든 것이 안정적인 쿠버네티스 환경 위에서 구동됩니다.

```mermaid
graph TD
    A[사용자 브라우저] --> B(프론트엔드: React/Tailwind);
    B --> C(백엔드 API: FastAPI);
    C --> D(데이터베이스: PostgreSQL);

    subgraph Kubernetes 클러스터 (containerd 런타임)
        C --- K1(백엔드 컨테이너);
        D --- K2(데이터베이스 컨테이너);
        B --- K3(프론트엔드 컨테이너);
    end

    K1 -- API 요청/응답 --> K3;
    K1 -- 데이터 쿼리/저장 --> K2;
```

## 🔄 사용자 시나리오 및 데이터 흐름 (BOM 등록 예시)
관리자가 새로운 배터리 부품 관계(BOM)를 등록하는 과정은 다음과 같은 흐름으로 진행됩니다. 사용자가 프론트엔드에서 정보를 입력하면, 이 정보는 백엔드를 거쳐 데이터베이스에 안전하게 저장됩니다.

```mermaid
flowchart TD
    A[관리자] -- 1. "BOM 등록" 버튼 클릭 --> B(프론트엔드: BOM 등록 모달 오픈);
    B -- 2. 부모/자식 품목 및 수량 입력 후 "저장" 클릭 --> C{프론트엔드: API 요청 생성};
    C -- 3. HTTP POST /api/bom 요청 전송 --> D[백엔드: FastAPI 서버];
    D -- 4. 요청 데이터 유효성 검사 및 처리 --> E[데이터베이스: PostgreSQL];
    E -- 5. bom_relationships 테이블에 데이터 저장 --> F{백엔드: 저장 결과 반환};
    F -- 6. HTTP 200 OK 응답 및 성공 메시지 --> G[프론트엔드: 완료 알림 표시];
    G -- 7. 사용자에게 성공 메시지 표시 --> A;
```

## 💡 주요 기술 요약 (비전공자용 설명)

이 프로젝트는 효율적인 스마트 팩토리 시스템을 구축하기 위해 다음과 같은 주요 기술들을 사용하고 있습니다. 각 기술은 특정 역할을 담당하여 전체 시스템이 원활하게 운영되도록 돕습니다.

*   **인프라 (Kubernetes, containerd, PostgreSQL)**:
    시스템이 무너지지 않게 땅을 다지고 전기를 공급하는 역할입니다. 쿠버네티스는 마치 아파트 단지를 관리하는 시스템처럼, 백엔드와 프론트엔드 같은 각 프로그램(컨테이너)들이 독립적으로 잘 돌아가도록 배치하고 관리합니다. 컨테이너드는 이 프로그램들을 담는 상자(컨테이너)를 만들고 실행하는 기술이며, PostgreSQL은 모든 중요한 정보를 안전하게 보관하는 튼튼한 금고(데이터베이스)입니다.

*   **백엔드 (FastAPI, Python)**:
    요리사처럼 주문을 받아 실제로 데이터를 처리하는 두뇌 역할입니다. 사용자가 웹사이트에서 어떤 작업을 요청하면, 백엔드는 그 요청을 받아서 필요한 데이터를 데이터베이스에서 가져오거나, 새로운 데이터를 저장하는 등의 복잡한 계산과 처리를 수행합니다. 마치 주문을 받아 요리를 만들고 손님에게 제공하는 주방과 같습니다.

*   **프론트엔드 (React, Tailwind CSS, JavaScript)**:
    메뉴판처럼 사용자가 정보를 보고 주문하는 얼굴 역할입니다. 웹사이트에서 눈으로 보고 직접 클릭하고 입력하는 모든 부분이 프론트엔드입니다. 리액트는 이 웹사이트의 복잡한 화면을 효율적으로 구성하는 기술이고, 테일윈드 CSS는 웹사이트를 보기 좋게 꾸며주는 디자인 도구입니다. 이들은 사용자가 시스템을 쉽고 편리하게 이용할 수 있도록 아름답고 직관적인 인터페이스를 제공합니다.
