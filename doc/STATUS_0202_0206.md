📋 프로젝트 개요
프로젝트명: KNU Mobility MES (Manufacturing Execution System)

목표: 모빌리티 제조 공정의 디지털 트랜스포메이션을 위한 실시간 관제 및 관리 시스템 구축

주요 기술 스택: FastAPI (Backend), React (Frontend), PostgreSQL (Database), Kubernetes (Infrastructure)

🛠️ 개발 현황 및 성과
1. 인프라 및 시스템 안정성 (Infrastructure)
쿠버네티스 클러스터 최적화: 최신 표준인 containerd 런타임 기반으로 클러스터를 재구성하여 안정적인 컨테이너 오케스트레이션을 구현했습니다.

데이터 영속성 확보: PostgreSQL에 PV(Persistent Volume) 및 PVC(Persistent Volume Claim)를 적용하여 파드(Pod) 재시작이나 VM 부팅 시에도 데이터가 유실되지 않는 환경을 구축했습니다.

자동 복구 시스템: VM 재부팅 시 유동 IP를 자동으로 감지하고, 쿠버네티스 API 서버의 헬스체크 및 프론트엔드 연결 정보를 동기화하는 startup-fix.sh 스크립트를 개발하여 운영 편의성을 높였습니다.

2. 백엔드 개발 (Backend API)
FastAPI 기반 마이크로서비스: 품목 마스터(/api/data), 설비 상태, 그리고 최근 추가된 BOM 등록(/api/bom) 엔드포인트를 구현 완료했습니다.

데이터 모델링: items, equipments, boms 테이블 간의 관계형 DB 스키마 설계를 완료하고, 부모-자식 관계를 정의하는 비즈니스 로직을 반영했습니다.

3. 프론트엔드 및 UI/UX (Frontend)
하이엔드 대시보드: Tailwind CSS v4를 활용하여 다크 테마 기반의 전문가용 관제 UI를 구축했습니다.

실시간 모니터링: 5초 주기로 인벤토리 현황과 생산 라인 설비 가동 여부(RUN/IDLE)를 자동 갱신하여 보여줍니다.

BOM 관리 모달: 사용자가 화면에서 직접 부모 부품과 자식 부품을 선택하고 소요량을 입력할 수 있는 대화형 등록 시스템을 통합했습니다.
