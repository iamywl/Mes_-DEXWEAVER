## 1. 시스템 개요

KNU MES(Manufacturing Execution System)는 제조 현장의 생산 활동을 실시간으로 관리하는 통합 시스템입니다.

### 주요 기능
| 영역 | 기능 |
|------|------|
| 기준정보 | 품목 마스터, BOM, 공정/라우팅, 설비 관리 |
| 생산관리 | 생산계획 수립, 작업지시, 실적 수집, 진도 관리 |
| 품질관리 | 검사 기준 설정, 수입/공정/출하 검사, 불량 분석 |
| 재고관리 | 입고/출고, LOT 추적, 안전재고 경보 |
| AI 분석 | 수요 예측, 불량 예측, 설비 고장 예측, AI 인사이트 |
| 모니터링 | 네트워크 플로우, 인프라 상태, K8s 파드 관리 |

### 기술 스택
- **Frontend**: React 19 + Tailwind CSS (Nginx 서빙)
- **Backend**: FastAPI (Python 3.9)
- **Database**: PostgreSQL 15
- **Infra**: Kubernetes (K8s)

## 5. BOM 관리

BOM(Bill of Materials, 자재명세서)은 완제품/반제품을 만들기 위해 필요한 하위 자재 목록입니다.

### 조회
1. **BOM** 메뉴 클릭
2. 품목 코드를 입력하고 **전개** 버튼 클릭
3. 해당 품목의 BOM 트리가 재귀적으로 전개됨

### BOM 전개 예시

**ITEM015 (전자제어장치) 1개 생산 시 필요 자재:**

```
전자제어장치 (ITEM015) x 1
├── CNC가공부품 (ITEM010) x 1.0 (로스율 1.0%)
│   └── 알루미늄봉 (ITEM002) x 1.5kg (로스율 5.0%)
├── 사출성형품 (ITEM011) x 1.0 (로스율 0.5%)
│   └── 플라스틱펠릿 (ITEM004) x 0.8kg (로스율 2.0%)
├── 와이어하네스 (ITEM012) x 1.0 (로스율 1.0%)
│   ├── 구리와이어 (ITEM003) x 3.0m (로스율 1.5%)
│   └── 전자부품A (ITEM006) x 2.0ea (로스율 0.5%)
├── 전자부품A (ITEM006) x 5.0ea (로스율 0.5%)
└── 볼트너트세트 (ITEM007) x 8.0set (로스율 1.0%)
```

### 용어 설명
- **qty_per_unit**: 상위 품목 1개 만드는 데 필요한 하위 품목 수량
- **loss_rate (%)**: 생산 과정 중 손실률 (절삭칩, 불량 등)
- **소요량 계산**: `필요수량 = qty_per_unit × (1 + loss_rate/100) × 상위주문수량`

## 8. 생산계획 (Plans)

월별/주별 생산계획을 수립하고 진도를 관리합니다.

### 조회
1. **Plans** 메뉴 클릭
2. 전체 생산계획 목록이 테이블로 표시됨
3. 각 계획의 품목, 수량, 납기, 우선순위, 상태, 진척률 확인 가능

### 생산계획 상태
| 상태 | 의미 |
|------|------|
| WAIT | 대기 (아직 작업지시 미발행) |
| PROGRESS | 진행중 (작업지시 발행, 실적 수집 중) |
| DONE | 완료 (목표 수량 달성) |

### 우선순위
| 우선순위 | 의미 |
|----------|------|
| HIGH | 긴급 (납기 임박 또는 주요 고객) |
| MID | 보통 |
| LOW | 낮음 (여유 납기) |

### 현재 등록 계획 (25건)
- **완료**: 2025-11~2026-02 기간 중 13건
- **진행중**: 2026-02월 중 6건 (ITEM010, ITEM013, ITEM014, ITEM015, ITEM016)
- **대기**: 2026-02~03월 6건 (ITEM017, ITEM018, ITEM019, ITEM020)

## 9. 작업지시 (Work Order)

생산계획에 따라 현장에 내리는 구체적인 작업 지시를 관리합니다.

### 조회
1. **Work Order** 메뉴 클릭
2. 전체 작업지시 목록이 표시됨
3. 작업일자, 설비, 품목, 지시수량, 상태 확인 가능

### 작업지시 ID 규칙
```
WO-YYYYMMDD-SEQ
예) WO-20260210-001 → 2026년 2월 10일 첫 번째 작업지시
```

### 작업지시 상태
| 상태 | 의미 |
|------|------|
| WAIT | 대기 (아직 작업 미시작) |
| WORKING | 작업 중 (실적 수집 중) |
| DONE | 완료 |

### 작업 실적
각 작업지시에 대해 작업 실적이 기록됩니다:
- **양품수(good_qty)**: 정상 생산된 수량
- **불량수(defect_qty)**: 불량 발생 수량
- **불량코드**: DEF-VISUAL(외관), DEF-DIMEN(치수), DEF-FUNC(기능), DEF-ASSEMBLY(조립), DEF-ELEC(전기), DEF-PLATING(도금), DEF-WELD(용접), DEF-MATERIAL(소재)
- **작업자**: 실제 작업 수행자
- **시작/종료 시간**: 작업 소요 시간 추적

## 10. 품질관리 (Quality)

수입/공정/출하 검사 결과와 불량 분석을 관리합니다.

### 조회
1. **Quality** 메뉴 클릭
2. 불량 현황 요약과 추이 데이터가 표시됨

### 검사 유형
| 유형 | 시점 | 설명 |
|------|------|------|
| INCOMING | 수입검사 | 원자재/부품 입고 시 품질 확인 |
| PROCESS | 공정검사 | 생산 중 중간 품질 확인 |
| OUTGOING | 출하검사 | 완제품 출하 전 최종 확인 |

### 품질 검사 기준 (30개)
각 품목별로 검사 항목이 등록되어 있습니다:
- **NUMERIC**: 수치 측정 (길이, 무게, 전압 등) — 최소/최대 범위 내 합격
- **VISUAL**: 외관 검사 (스크래치, 변색 등) — 육안 판정
- **FUNCTIONAL**: 기능 검사 (동작 시험) — 기능 합격/불합격

### 불량 코드 (8종)
| 코드 | 명칭 | 설명 |
|------|------|------|
| DEF-VISUAL | 외관불량 | 스크래치, 변색, 찍힘 |
| DEF-DIMEN | 치수불량 | 규격 치수 이탈 (공차 초과) |
| DEF-FUNC | 기능불량 | 작동 불량, 성능 미달 |
| DEF-MATERIAL | 소재불량 | 원자재 기포, 크랙 |
| DEF-ASSEMBLY | 조립불량 | 오조립, 나사 풀림 |
| DEF-PLATING | 도금불량 | 도금 박리, 얼룩 |
| DEF-WELD | 용접불량 | 비드 불량, 기공 |
| DEF-ELEC | 전기불량 | 단선, 합선 |

## 11. 재고관리 (Inventory)

원자재/반제품/완제품의 재고를 LOT 단위로 관리합니다.

### 조회
1. **Inventory** 메뉴 클릭
2. 전체 재고 현황이 테이블로 표시됨
3. 품목코드, LOT번호, 수량, 창고, 위치, 안전재고 상태 확인

### 재고 상태 표시
| 상태 | 색상 | 의미 |
|------|------|------|
| NORMAL | 녹색 | 재고 충분 (안전재고 이상) |
| LOW | 황색 | 재고 부족 (안전재고 미만) |
| OUT | 적색 | 재고 없음 (0) |

### 창고 구분
| 창고 | 용도 | 보관 품목 |
|------|------|-----------|
| WH01 | 원자재 창고 | 원자재 (ITEM001~ITEM008) |
| WH02 | 반제품 창고 | 반제품 (ITEM009~ITEM014) |
| WH03 | 완제품 창고 | 완제품 (ITEM015~ITEM020) |

### 위치(Location) 규칙
```
A-01-01 형식: [존]-[열]-[단]
A존: 원자재, B존: 반제품, C존: 완제품
```

### LOT 번호 규칙
```
LOT-YYYYMMDD-SEQ
예) LOT-20260201-001 → 2026년 2월 1일 첫 번째 LOT
```

### 재고 이동 유형

| IN | 입고 (구매 입고 또는 생산 완료 입고) |

## 12. AI 센터 (AI Center)

AI 기반의 예측 분석 기능을 제공합니다. 3가지 예측 모델을 사용할 수 있습니다.

### 12.1 수요 예측 (Demand Forecast)

과거 출하 이력 데이터를 기반으로 향후 수요를 예측합니다.

**사용 방법:**
1. **AI Center** 메뉴 클릭
2. **수요예측** 패널에서 품목 코드 입력 (예: `ITEM015`)
3. 예측 개월 수 입력 (예: `3`)
4. **예측실행** 버튼 클릭

**결과 해석:**
- 월별 예측 수량이 리스트로 표시됨
- 선형회귀 모델 기반으로 과거 24개월 출하 추세를 분석하여 예측

**예측 가능 품목 (출하 이력 보유):**
- ITEM015 (전자제어장치) — 24개월 출하 데이터
- ITEM016 (유압밸브) — 24개월
- ITEM017 (센서모듈) — 24개월
- ITEM018 (모터드라이버) — 18개월
- ITEM019 (산업용커넥터) — 12개월
- ITEM020 (제어패널) — 12개월

### 12.2 불량 예측 (Defect Prediction)

생산 환경 조건(온도, 압력, 속도, 습도)을 입력하면 예상 불량률을 예측합니다.

**사용 방법:**
1. **불량예측** 패널에서 파라미터 입력:
   - 온도 (℃): 공정 온도 (정상: 180~210, 위험: 230+)
   - 압력 (MPa): 공정 압력 (정상: 8~12, 위험: 14+)
   - 속도 (rpm): 공정 속도 (정상: 45~55, 위험: 60+)
   - 습도 (%): 환경 습도 (정상: 50~60, 위험: 75+)
2. **예측실행** 버튼 클릭

**결과 해석:**
- 예측 불량률(%) 표시
- 위험 요소별 상세 분석
- 판정: NORMAL (< 3%), WARNING (3~5%), DANGER (> 5%)

**예시:**
```
온도=230, 압력=14, 속도=60, 습도=75 → 예측 불량률: 8.2% (DANGER)
온도=195, 압력=10, 속도=50, 습도=55 → 예측 불량률: 1.5% (NORMAL)
```

### 12.3 설비 고장 예측 (Failure Prediction)

설비 센서 데이터(진동, 온도, 전류)를 입력하면 고장 확률을 예측합니다.

**사용 방법:**
1. **고장예측** 패널에서 파라미터 입력:
   - 진동 (mm/s): 정상 < 2.0, 경고 2.0~3.5, 위험 > 3.5
   - 온도 (℃): 정상 < 50, 경고 50~65, 위험 > 65
   - 전류 (A): 정상 < 13, 경고 13~16, 위험 > 16
2. **예측실행** 버튼 클릭

**결과 해석:**
- 고장 확률(%) 표시
- 각 요소(진동/온도/전류)별 기여도 분석
- 0~30%: 정상, 30~60%: 주의, 60~80%: 경고, 80%+: 긴급

**예시:**
```
진동=3.5, 온도=65, 전류=16 → 고장확률: 62.5% (경고)
진동=0.8, 온도=38, 전류=10 → 고장확률: 5.0% (정상)
```

## 13. 리포트 (Reports)

생산 실적과 품질 데이터를 종합 분석한 리포트를 제공합니다.

### 13.1 생산 리포트

**표시 정보:**
- **기간별 생산 추이**: 일/주/월 단위 양품수, 불량수, 생산 수량 집계
- **품목별 실적**: 각 품목의 양품수, 불량수, 불량률
- **달성률**: (양품수 / 계획수량) × 100

**사용 방법:**
1. **Reports** 메뉴 클릭
2. 생산 리포트 섹션에서 기간별 데이터 확인
3. 진도 바(Progress Bar)로 달성률 시각 확인

### 13.2 품질 리포트

**표시 정보:**
- **전체 불량률**: 불량수 / 총 생산수 × 100
- **불량 추이**: 일별 불량률 변화
- **Cpk 근사치**: 공정능력지수 추정값

### 13.3 AI 인사이트

**Insights** 버튼을 클릭하면 AI가 현재 시스템 상태를 분석하여 이슈와 개선 권고사항을 제시합니다.

**분석 항목:**
- 생산 달성률 < 90% 경고
- DOWN 상태 설비 감지
- 불량률 > 3% 품목 경고
- 개선 권고 사항 제시

## 18. FAQ / 문제 해결

### Q1. 사이트가 로드되지 않습니다
- 서버 IP와 포트(30173) 확인
- K8s 파드 상태 확인: `kubectl get pods`
- 프론트엔드 파드가 Running 상태인지 확인

### Q2. 데이터가 표시되지 않습니다
- 백엔드 API 파드 상태 확인: `kubectl get pods | grep mes-api`
- DB 연결 확인: `kubectl exec deployment/postgres -- psql -U postgres -d mes_db -c "SELECT 1"`
- 브라우저 개발자 도구(F12) → Network 탭에서 API 오류 확인

### Q3. AI 예측이 동작하지 않습니다
- 수요 예측: 해당 품목의 출하 이력이 최소 3개월 이상 필요
- 불량/고장 예측: 숫자 값을 올바르게 입력했는지 확인

### Q4. 재고가 실제와 다릅니다
- 재고 데이터는 시드 데이터로 초기화된 상태
- 입고/출고 API를 통해 실시간 반영 필요

### Q5. 설비 상태를 변경하고 싶습니다
- API 호출: `PUT /api/equipment/{equip_code}/status`
- Body: `{"status": "RUNNING|STOP|DOWN", "reason": "사유"}`
