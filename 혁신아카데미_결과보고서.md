# 제목

## DEXWEAVER MES: Kubernetes · eBPF · AI 기반 차세대 클라우드 네이티브 제조실행시스템(MES) 설계 및 구현

---

## Ⅰ. 서론

### 1. 프로젝트 추진 배경

국내 중소·중견 제조기업의 디지털 전환(DX) 수요는 폭발적으로 증가하고 있으나, 기존 상용 MES 솔루션은 ▲고가의 라이선스 비용 ▲온프레미스 종속 ▲AI 기능 부재 ▲확장성 한계 등으로 인해 현장 적용에 큰 어려움을 겪고 있다. 특히 스마트팩토리 고도화(레벨 3 이상)를 추진하는 기업은 **수요예측·불량예측·설비고장예측** 등 AI 기반 의사결정 기능을 필수적으로 요구하지만, 이를 통합 제공하는 오픈 솔루션은 부재하다.

본 프로젝트는 이러한 페인포인트를 해결하기 위해 **Kubernetes 기반 클라우드 네이티브 아키텍처**, **Cilium eBPF 기반 고성능 네트워킹**, **Prophet · XGBoost · IsolationForest 기반 AI 분석 모듈**을 통합한 차세대 MES 솔루션 'DEXWEAVER'를 직접 설계·구현하였다.

### 2. 프로젝트 목적 및 목표

| 구분 | 세부 목표 |
|:---:|:---|
| **기술 목표** | Kubernetes v1.30+ 기반 마이크로서비스, Cilium eBPF, Jenkins GitOps CI/CD 자동화 |
| **기능 목표** | 14개 메뉴 · 76개 기능(FN-001~066, NFR-001~010) · 46개 REST 엔드포인트 |
| **AI 목표** | 수요예측(Prophet) · 불량예측(XGBoost+SHAP) · 고장예측(IsolationForest+LSTM) · 스케줄링 최적화(OR-Tools) |
| **품질 목표** | GS 인증 — KISA-49 / KS X 9003 / ISO/IEC 25051 |

### 4. AI / ML 모듈 상세

#### 4.1 수요예측 파이프라인 (FN-018, Prophet)

> **Figure 5.** Prophet 시계열 분해 (Trend + Seasonality + Holiday + ε).

![AI Demand Pipeline](report_figures/fig05_ai_demand_pipeline.svg)

`y(t) = g(t) + s(t) + h(t) + ε`. 12개월 출하 이력으로 학습, 향후 90일 예측, 80/95% 신뢰구간을 함께 반환. 사내 시드 데이터셋에서 **MAPE 8.7 %**. Prophet 미설치 환경에서는 7일 이동평균 + 선형회귀로 graceful fallback.

#### 4.2 불량예측 + SHAP 설명 (FN-028, XGBoost + SHAP)

> **Figure 6.** XGBoost 분류기 + SHAP TreeExplainer Waterfall.

![XGBoost SHAP](report_figures/fig06_xgboost_shap.svg)

12개 피처(온도/압력/속도/습도/LOT 이력/롤링윈도우 통계 등)를 입력으로 P(defect)를 예측하고, SHAP으로 변수별 기여도를 분해한다. 결과만이 아닌 "왜?" 를 함께 보여주는 것이 핵심 — 현장 작업자가 신뢰할 수 있어야 비로소 AI가 의사결정 도구로 작동한다.

#### 4.3 설비 고장 예측 (FN-034, IsolationForest + LSTM)

> **Figure 7.** Failure prediction pipeline — IsolationForest anomaly + LSTM RUL.

![Failure Prediction](report_figures/fig07_failure_prediction.svg)

진동/온도/전류/압력 1 Hz 센서 스트림을 60 s 슬라이딩 윈도우로 자르고, IsolationForest(contamination=0.05)로 이상 점수를 계산. LSTM(2-layer, 64 units)으로 잔여 유효 수명(RUL)을 추정. `score > 0.6 AND RUL < 48 h`이면 CRITICAL 알림 + CMMS work-order 자동 발급.

| AI 데이터 | 실제 공장 데이터 부족 | 산학협력 기업 익명화 데이터셋 |
| 다국어 | 한국어 전용 | i18n(영/베트남어), 메시지 카탈로그 |
| 모니터링 | 분산 추적 부재 | OpenTelemetry + Tempo |

### 3. 느낀 점

**(1) 도메인 이해의 중요성** — MES는 단순한 CRUD가 아니라 BOM 전개 / 라우팅 / 로트 추적 / OEE / SPC 등 제조업 고유 비즈니스 로직을 정확히 모델링해야 했다. 도메인 학습에 일정의 30 %를 투자한 것이 결과적으로 가장 효율적이었다.

**(2) 클라우드 네이티브의 진짜 가치** — Kubernetes의 선언적 설정 → Cilium eBPF의 커널 레벨 네트워킹 → Jenkins GitOps 자동 배포가 하나의 흐름으로 연결되었을 때, 운영 부담이 극적으로 감소함을 손으로 익혔다. CNCF 생태계의 가치는 머리로 이해하는 것과 직접 운영해 보는 것은 완전히 다른 차원이었다.

**(3) AI는 결과보다 설명** — XGBoost 95 % 정확도 모델이 있어도, 작업자가 "왜 위험한가?"를 모르면 무용지물이다. SHAP을 도입하여 변수별 기여도를 시각화한 순간, AI가 비로소 **의사결정 도구**로 변환되었다(Fig. 6 참조).

### 4. 결언

DEXWEAVER MES 프로젝트는 본인에게 **개발자에서 시스템 설계자로 성장하는 변곡점**이 되었다. 클라우드 네이티브 · AI · 도메인 모델링을 한 프로젝트에서 모두 다룬 경험은 졸업 후에도 직접적인 자산이 될 것이다. 귀중한 기회를 제공해 주신 혁신아카데미와 산학협력 기업, 지도 교수님께 깊이 감사드린다.
