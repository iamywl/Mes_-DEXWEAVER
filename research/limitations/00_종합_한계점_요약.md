## 5. 리서치 데이터 신뢰도 요약

| 보고서 | 신뢰도 | 주요 한계 |
|--------|--------|----------|
| 01 MES 개요 | 높음 | ISA-95/MESA 공식 자료 기반 |
| 02 시장규모 | 중간 | 기관별 최대 62% 편차 ($25.8B vs $41.8B) |
| 03 경쟁사 분석 | 중간 | 공개 정보만 활용, 실제 점유율 비공개 |
| 04 기능 비교 | 중간 | PoC 없이 마케팅 자료 기반 비교 |
| 05~07 심층 비교 | 중간~낮음 | 독립적 벤치마크 미검증 |

3. **파일럿 PoC 실시** (장기 6~12개월)

## 6. 종합 개선 로드맵

### Phase 1: 핵심 기능 보완 (1~3개월)

| 항목 | 설명 | 기대 효과 |
|------|------|----------|
| DB 인덱스 추가 | FK 컬럼, 자주 조회하는 컬럼에 인덱스 | 쿼리 성능 3~10배 향상 |
| 프론트엔드 모듈화 | App.jsx를 14개 컴포넌트 파일로 분리 | 유지보수성 대폭 개선 |
| OEE 자동 계산 | equip_status_log + work_results 기반 | MESA-11 성과분석 충족 |
| SPC 관리도 | X-bar, R chart, Cpk 계산 | 품질관리 수준 향상 |
| Docker 이미지 빌드 | ConfigMap 배포 → 컨테이너 이미지 | 배포 안정성/속도 개선 |
| 인증 미들웨어 | 반복 코드 제거, FastAPI Depends 사용 | 코드 품질 향상 |

### Phase 2: 고급 기능 추가 (3~6개월)

| 항목 | 설명 | 기대 효과 |
|------|------|----------|
| CAPA 프로세스 | 시정/예방 조치 워크플로우 | 품질 관리 체계 완성 |
| MQTT 데이터 수집 | 설비 센서 자동 연동 | 실시간 데이터 확보 |
| CMMS 유지보수 | 예방보전 일정, 정비이력 | 설비 관리 체계 완성 |
| AI 모델 캐싱 | 학습된 모델 저장/재사용 | API 응답시간 10배 개선 |
| 레시피 관리 | ISA-88 기반 레시피/파라미터 | 공정 관리 수준 향상 |

### Phase 3: 엔터프라이즈 전환 (6~12개월)

| 항목 | 설명 | 기대 효과 |

| OPC-UA 연동 | 설비 직접 통신 | ISA-95 Level 2 연동 |
| ERP 인터페이스 | SAP/Oracle 연동 API | ISA-95 Level 4 연동 |
| DMS (문서관리) | 작업지시서/검사성적서 | 문서 관리 기능 확보 |
| 다국어 지원 | i18n 프레임워크 | 글로벌 시장 진출 |
| 디지털 트윈 | 3D 모델 연계 | 차세대 MES 기능 |

## 7. DEXWEAVER 차별화 전략 제안

### 현재 강점 (경쟁사 대비 우위)
1. **AI 네이티브 통합**: Prophet(수요), XGBoost/SHAP(불량), IsolationForest(설비), OR-Tools(스케줄링) — 4개 AI 모델 기본 탑재
2. **클라우드 네이티브**: K8s + Cilium eBPF 기반, 컨테이너 오케스트레이션 기본
3. **오픈소스 기반**: 라이선스 비용 Zero, 커스터마이징 자유도 높음

### 권장 포지셔닝
> **"AI-First 클라우드 네이티브 경량 MES"**

- 경쟁사(Siemens, SAP 등)의 무거운 엔터프라이즈 MES 대비
- AI 기반 예측(수요/불량/고장) + 최적화(스케줄링)를 핵심 차별점으로
- 중소기업 대상 빠른 도입(< 4주) + 낮은 TCO를 강조
